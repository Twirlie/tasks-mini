## Agent Brief

**Category:** enhancement
**Summary:** Extract the ColumnView UI component from the main app component into a dedicated component file.

**Current behavior:**
The `ColumnView` component (column rendering, task addition form, and drag & drop integration) is defined inline in the main app component. It cannot be tested or developed in isolation.

**Desired behavior:**
A dedicated `column_view.rs` file in the `components` module contains the `ColumnView` component. It accepts a `Column` and its tasks, renders the column header, displays tasks via `TaskCard`, and handles new task creation via the `create_task` IPC command. The main app component imports `ColumnView` through the components module re-export.

**Key interfaces:**
- `ColumnView` — Leptos component accepting `Column` and `Vec<Task>` props, plus a board refresh callback
- Renders column name, list of tasks via `DropZone` + `DraggableTask` + `TaskCard`, and an add-task form
- Add-task form calls `invoke("create_task", {title, description, columnId})` then updates local board state
- Uses `DropZone` (from utils module) for each task position to enable drag & drop reordering

**Acceptance criteria:**
- [ ] `column_view.rs` contains the `ColumnView` component with correct props
- [ ] `components/mod.rs` re-exports `ColumnView`
- [ ] The main app component imports `ColumnView` from the components module
- [ ] Component tests verify ColumnView renders with column and tasks
- [ ] Component tests verify task addition form triggers create_task IPC correctly
- [ ] The application compiles and ColumnView renders correctly after refactoring

**Out of scope:**
- Changing ColumnView UI or behavior
- Adding new features like column deletion or renaming
- Moving the main App component (handled in Issue #5)
- Backend changes
