## Agent Brief

**Category:** enhancement
**Summary:** Extract the TaskCard UI component from the main app component into a dedicated component file.

**Current behavior:**
The `TaskCard` component (including task rendering and delete functionality) is defined inline in the main app component. It cannot be tested or developed in isolation.

**Desired behavior:**
A dedicated `components` module exists with a `task_card.rs` file containing the `TaskCard` component. The component accepts a `Task` and renders it with a delete button that calls the `delete_task` IPC command. The main app component imports `TaskCard` through the components module re-export.

**Key interfaces:**
- `TaskCard` — Leptos component accepting a `Task` prop
- Renders task title, optional description, and a delete button
- Delete button calls `invoke("delete_task", {id: task_id})` then refreshes board state by calling `invoke("get_board")`
- The component must remain wrapped by `DraggableTask` (which is provided by the utils module from Issue #2)

**Acceptance criteria:**
- [ ] A `components` module exists with `mod.rs` re-exporting components
- [ ] `task_card.rs` contains the `TaskCard` component with correct props
- [ ] The main app component imports `TaskCard` from the components module
- [ ] Component tests verify TaskCard renders with task data
- [ ] Component tests verify delete button triggers the correct IPC flow
- [ ] The application compiles and TaskCard renders correctly after refactoring

**Out of scope:**
- Changing TaskCard UI or behavior
- Adding new features like task editing
- Moving ColumnView or other components
- Backend changes
