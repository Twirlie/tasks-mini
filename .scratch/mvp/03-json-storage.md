## Agent Brief

**Goal:** Create the `json_storage` module in `src-tauri/src/json_storage/` implementing the `Storage` trait with JSON file persistence.

**Key behaviors:**
- Data directory: `~/.local/share/tasks-mini/workflow/` (use `dirs::data_dir()` + `tasks-mini/workflow`)
- Main file: `tasks.json` containing the full `Board` serialized as JSON
- Schema versioning: `Board.schema_version` field in JSON
- Auto-backup: copy `tasks.json` to `tasks.json.backup.<timestamp>` before every write
- Backup rotation: keep last 5 backups, delete older ones
- Corrupt file handling: return `StorageError::Serialization` if JSON parse fails

**Implementation notes:**
- Implement `Storage` trait from `crate::storage_port`
- Use `tokio::fs` for async file operations (Tauri uses tokio runtime)
- Create data directory on init if it doesn't exist
- Use `chrono::Utc::now().timestamp()` for backup timestamps
- On `load_board()`: if file doesn't exist, return a default board with 4 columns (Backlog, Todo, In Progress, Done)
- On `save_board()`: backup first, then write atomically (write to temp file, rename)
- `create_task`/`update_task`/`delete_task`: load board, mutate, save board
- Same pattern for column operations

**Dependencies to add:**
- `dirs = "6"`
- `tokio = { version = "1", features = ["fs"] }`

**Tests:** Integration tests using `tempfile::TempDir`. Test read/write roundtrip, backup creation and rotation, schema versioning, corrupt file error, default board creation on missing file.
