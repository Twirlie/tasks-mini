## Agent Brief

**Goal:** Implement drag-and-drop in the Leptos frontend for moving tasks between columns and reordering within columns.

**What to build:**
- Drag-and-drop using the HTML5 Drag and Drop API (no external JS library needed for MVP)
- Each task card is `draggable="true"`
- Each column is a drop zone
- On drag start: store task id and source column id in `DataTransfer`
- On drop on a column: call `move_task` Tauri command with target column_id and new order
- On drop for reorder within same column: update task order field

**Implementation notes:**
- Frontend module in `src/` (Leptos components)
- Use Leptos `create_signal` for drag state tracking
- Call `invoke("move_task", ...)` via `window.__TAURI__.core.invoke`
- After successful drop, refresh board state from backend
- Smooth animations: use CSS `transition` on task cards for position changes
- Visual feedback: add CSS class `drag-over` to highlight drop zones, `dragging` class on dragged element
- Prevent drop on invalid areas (e.g., outside columns)

**Edge cases:**
- Drop on same position → no-op
- Concurrent modifications → refresh board after each drop
- Invalid drop area → ignore event

**Dependencies:** None new — uses existing Leptos + Tauri invoke API

**TDD Cycles** (execute one at a time, RED→GREEN→REFACTOR):
1. `drag start stores task id and source column in DataTransfer` → drag event handler → no refactor

3. `drop within same column updates task order` → reorder drop → no refactor
4. `drop on same position is a no-op` → same-position guard → no refactor
5. `board state refreshes after successful drop` → post-drop refresh → no refactor
6. `drop zone highlights on drag-over` → CSS drag-over class → no refactor
7. `dragged card gets visual feedback` → CSS dragging class → no refactor
